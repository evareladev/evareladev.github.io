NumPy es una biblioteca de código abierto de Python diseñada para gestionar grandes matrices y arreglos multidimensionales de datos numéricos, así como para realizar operaciones matemáticas con estas estructuras. Fue creada por Travis Oliphant en 2005 y desde entonces se ha convertido en una parte integral del ecosistema de ciencia de datos de Python. NumPy utiliza estructuras llamadas arrays que permiten realizar operaciones matemáticas más rápidas y con menos código que usando listas tradicionales.

1.2. Por qué es importante NumPy?

  • Manejo eficiente de datos: NumPy proporciona una forma eficiente de almacenar y manipular datos, lo que lo hace ideal para tareas como la limpieza y el preprocesamiento de datos.
  • Velocidad: NumPy está escrito en C, lo que le otorga una ventaja significativa en velocidad sobre Python puro al realizar cálculos numéricos.
  • Operaciones con arreglos: NumPy ofrece una amplia gama de funciones matemáticas para realizar operaciones con arreglos, como la suma, la resta y la multiplicación elemento por elemento.

1.4. Creación de matrices en NumPy

NumPy trabaja principalmente con matrices. Puedes crear matrices de varias maneras:

  1. Crear una matriz de ceros

print(zeros)
  1. Crear una matriz de unos

print(ones)

1.6. Broadcasting

El Broadcasting permite realizar operaciones con matrices de diferentes formas sin modificarlas explícitamente. Simplifica el código y lo hace más conciso. Así funciona la difusión:

1.6.1. Reglas de Broadcasting.

  • Si dos matrices tienen un número diferente de dimensiones, la forma de la matriz de menor dimensión se rellena con unos en el lado izquierdo.
  • Si la forma de las dos matrices no coincide en ninguna dimensión, la matriz con forma igual a 1 en esa dimensión se estira para que coincida con la forma de la otra matriz. Si no se cumple ninguna de estas condiciones, NumPy genera un error "ValueError".

1.6.2. jemplos de Broadcasting

Ejemplo 1: Broadcasting con un escalar

arr = np.array([1, 2, 3])
result = arr + 5

En este ejemplo, NumPy difunde el escalar 5 para que coincida con la forma del array arr, lo que resulta en [6, 7, 8].

Ejemplo 2: Difusión de matrices unidimensionales y bidimensionales*

arr1 = np.array([1, 2, 3])
arr2 = np.array([[10], [20], [30]])
result = arr1 + arr2

Aquí, NumPy difunde la matriz unidimensional arr1 para que coincida con la forma de la matriz bidimensional arr2, lo que resulta en una matriz bidimensional [[11, 12, 13], [21, 22, 23], [31, 32, 33]].

El Broadcasting simplifica muchas operaciones, eliminando la necesidad de bucles explícitos o la remodelación de matrices.

1.7. Técnicas avanzadas de NumPy

NumPy ofrece técnicas avanzadas para la manipulación eficiente de datos. Comprender estas técnicas puede mejorar significativamente la lógica de desarrollo sobretodo en áreas como la ciencia de datos e IA.

1.7.1. Segmentación e indexación

NumPy proporciona potentes mecanismos para segmentar e indexar matrices, lo que le permite acceder fácilmente a elementos o secciones específicas de una matriz.

Ejemplo de segmentación básica:

arr = np.array([1, 2, 3, 4, 5])
slice = arr[1:4]
# Recupera los elementos 2, 3 y 4

# Indexación booleana
arr = np.array([1, 2, 3, 4, 5])
mask = arr > 2
result = arr[mask]
# Recupera los elementos donde la condición es verdadera: [3, 4, 5]

Indexación de matrices enteras:

arr = np.array([1, 2, 3, 4, 5])
indices = np.array([0, 2, 4])
result = arr[indices] 
# Recupera los elementos en los índices especificados: [1, 3, 5]

1.7.2. Concatenación y división

NumPy permite concatenar y dividir matrices a lo largo de ejes específicos.

#Concatenación
arr1 = np.array([1, 2, 3])
arr2 = np.array([4, 5, 6])
concatenated = np.concatenate((arr1, arr2))

#División
arr = np.array([1, 2, 3, 4, 5, 6])
split = np.split(arr, 3) # Divide la matriz en 3 partes iguales

Normalización de datos

La normalización global es una técnica fundamental que nos permite reescalar todos nuestros valores de consumo energético a un rango común entre 0 y 1. Este proceso es particularmente útil cuando:

  • Necesitamos comparar magnitudes diferentes al llevar todos los datos a la misma escala, podemos identificar patrones que serían difíciles de detectar con las magnitudes originales.
  • Preparamos datos para algoritmos de machine learning, esto porque muchos modelos funcionan mejor cuando los datos están normalizados.
  • Visualizamos tendencias con mayor claridad al eliminar las diferencias de escala, los gráficos muestran relaciones más puras entre variables.

Esta transformación es especialmente valiosa cuando queremos analizar patrones de comportamiento relativo más que valores absolutos de consumo.
